# Planejamento de Sprints - AgileLog

## Sprint 1 – Estrutura Inicial
- Criar repositório no GitHub
- Criar README.md com descrição do projeto
- Definir o backlog inicial
- Criar o quadro Kanban (A Fazer, Em Progresso, Concluído)
- Configurar o ambiente local e arquivos base

## Sprint 2 – CRUD de Tarefas
- Implementar CRUD (criar, listar, editar e excluir tarefas)
- Testar manualmente as funcionalidades
- Fazer commits com mensagens semânticas

## Sprint 3 – Testes Automatizados e CI
- Escrever testes automatizados com PyTest
- Criar workflow do GitHub Actions
- Validar execução automática de testes

## Sprint 4 – Mudança de Escopo
- Implementar nova funcionalidade: prioridade das tarefas
- Atualizar README.md explicando a mudança
- Atualizar o Kanban e o backlog
